Flow map

Prototype review should validate operational flows, not just static screens.

1. EnterLogin/register should feel calm and low-friction. No marketing noise.
2. OrientTask list answers “What matters now?” within 5 seconds.
3. ActCreate/edit flows reduce hesitation with clear field grouping and low-noise validation.
4. ConfirmDestructive actions are confirmed. Success states are clear but lightweight.

Scenario review + motion posture

Scenarios help the PO evaluate whether the system handles real usage patterns, not just ideal screenshots.

Scenario A · Busy IC
Open app, scan in-progress tasks, reprioritize one item, jump to detail, return fast.
Scenario B · Team lead
Create a task, assign owner, set due date, verify it appears in active list with clear status.
Scenario C · Recovery
No tasks or API hiccup must still give a clear path forward through empty/error states.
